## 快速开始

### 1. 环境准备

在 Postman 中设置环境变量：

```json
{
  "baseUrl": "http://localhost:8080",
  "token": "your-login-token"
}
```

### 2. 登录获取 Token

首先调用登录接口获取 token：

```
POST {{baseUrl}}/api/system/login
Content-Type: application/json

{
  "username": "admin",
  "password": "123456"
}
```

将返回的 token 保存到环境变量中。

### 3. 导入接口集合

每个文档底部都包含 Postman 导入格式的 JSON，可以：
1. 复制 JSON 内容
2. 在 Postman 中选择 Import → Raw text
3. 粘贴 JSON 内容
4. 点击 Continue → Import

### 4. 使用接口

导入后，所有接口会自动使用环境变量中的 `baseUrl` 和 `token`，可以直接发送请求测试。

## 认证说明

### Token 认证

除特别说明外，大部分接口需要在请求头中携带 token：

```
sa-token: {{token}}
```

### 权限说明

系统采用基于角色的访问控制（RBAC）：

**系统角色**:
- `role:super:admin`: 超级管理员（拥有所有权限）
- `role:simple:admin`: 普通管理员（拥有大部分管理权限）
- `bangumi:rss:subscriber`: RSS订阅者（可以使用RSS相关功能）

**权限验证**:
- 需要认证的接口会返回 401 错误
- 无权限访问的接口会返回 403 错误

## 通用规范

### 请求格式

**JSON 请求**:
```
Content-Type: application/json
```

**表单上传**:
```
Content-Type: multipart/form-data
```

### 响应格式

所有接口统一返回以下结构：

```json
{
  "code": 200,      // 响应码
  "msg": "操作成功", // 响应消息
  "data": {}        // 响应数据
}
```

### 常见响应码

| 响应码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 401 | 未登录或 token 失效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 分页参数

分页接口统一使用以下参数：

```json
{
  "pageNo": 1,      // 页码，从1开始
  "pageSize": 10    // 每页条数
}
```

分页响应格式：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "pageNo": 1,
    "pageSize": 10,
    "total": 100,
    "pages": 10,
    "records": []
  }
}
```
